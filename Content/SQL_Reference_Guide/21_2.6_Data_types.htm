Boolean literals can be written as TRUE and FALSE or 1 and 0 respectively,
but are always displayed as 1 and 0 in the native client.

Examples:

CREATE  TABLE  boolean_values  (col1  bool);  
INSERT  INTO  boolean_values  VALUES  ((true),(false));

Time zones are not supported.
Milliseconds are stored as 3 digits. If milliseconds are ingested with more than 3 digits, the system will round to 3 digits.

String data types

Type Description Maximum size (not null) Synonym

VARCHAR(n)

String of ASCII characters at the length of n

n bytes

CHARACTER VARYING

NVARCHAR(n)

String of UNICODE characters at the length of n

4*n bytes

  • VARCHAR is right-padded with spaces. These trailing spaces are ignored when used in functions.

  • NVARCHAR supports multiple languages with UTF8 encoding.

  • VARCHAR/NVARCHAR column size cannot exceed the SQream max row length (100,000 characters).

  • Neither VARCHAR nor NVARCHAR supports the ASCII character 0 (=NULL).

Please note the following NVARCHAR restrictions:

  • NVARCHAR columns cannot be used as group-by keys where there are multiple count distincts in the query.

  • NVARCHAR cannot be used in a SELECT statement together with window function expressions ( "select x,WINDOW from ..." where x is an NVARCHAR ).

  • Regular expression functions (rlike, not rlike, regexp_count, regexp_substr, regexp_instr) are not supported

  • LIKE with NVARCHAR supports only the following patterns: "%abc", "%abc%" and "abc%".

  • NVARCHAR is not supported as a join key.
    As a workaround, customers are advised to use the CRC64 function (crc64(nvarchar_column)= crc64(nvarchar_column))

  • Add column with a non-empty NVARCHAR value is not supported

  • Casts between NVARCHAR and other types are not supported

  • Aggregate functions on NVARCHAR (other than count) are not supported

  • Scalar functions are not supported (chr, patindex, isprefixof, issuffixof, isinfixof, comparison operators (>,>=,<,<=)).
