Improvements

  • Better performance with concurrent I/O readers.
  • We’ve revamped our parallel I/O algorithms to get better performance when reading very large tables.
  • Better error messages when an invalid syntax is used to create tables, such as when using VARCHAR with no length specified
  • Better reflection of instance status when using the load balancer
  • We’ve added a flag to allow SQream DB’s CSV parser to skip rows containing newlines, even when the field is quoted. This is useful for several of our customers experiencing CSV parsing issues. This flag is named ‘hardRowDelimiter’ and is disabled by default.
  • Statement tree information (show_node_info) can now be viewed from any instance in the cluster
  • We’ve renamed some statement tree chunk-producer names to better reflect their role in the query.
