Configuring the OS language and timezone

You must verify that the OS language is set to English, that the correct timezone is configured, and that NTP is synchronized.

  • Configuring the OS locale
  • Configuring the timezone
  • Configuring NTP synchronization

Configuring the OS locale

To check the system locale specified on your CentOS VPS:

  1. Read the /etc/locale.conf file

    2. # cat /etc/locale.conf

  2. Find the line:

    3. LANG=

    It should read:

    LANG=en_US.utf8

If it's not en_US.utf8, set it to that value:

# sudo localectl set-locale LANG=en_US.UTF-8

Configuring the timezone

Run the following command to display all the available timezones:

# timedatectl list-timezones

This will give you a list of the timezones available for your server. When you find the region/timezone setting that is correct for your server, run the following command to set it:

# sudo timedatectl set-timezone region/timezone

Example

To set the timezone to United States Eastern time, type:

# sudo timedatectl set-timezone America/New_York

Configuring NTP synchronization

NTP synchronization allows your computer to stay in sync with other servers, optimizing predictability in operations that rely on having the correct time.

To synchronize NTP do the following:

  1. Install the ntp service from the CentOS default repositories:

    2. # sudo yum install -y ntp

  2. Start the service for this session.

    3. # sudo systemctl start ntpd

  3. Enable the service so that it is automatically started each time the server boots:

    4. # sudo systemctl enable ntpd

  4. Configure NTP service to sync against your NTP server or an Internet-based one. Your server will now automatically correct its system clock to align with the global servers.
